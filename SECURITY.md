# Security Policy

MoonTale is an early-stage web prototype for personalized bedtime stories. Security, privacy, and child-safety concerns should be handled carefully and privately.

## Reporting a Security Issue

Please report security issues privately by emailing:

```text
contact@moontaleapp.com
```

Do not publish security vulnerabilities, privacy concerns, or child-safety reports in public issues before they have been reviewed.

## Child Data and Privacy

Do not submit real child data in public issues, discussions, pull requests, screenshots, logs, or examples.

Avoid sharing:

- Full child names
- Email addresses
- Location details
- Private family details
- Screenshots containing personal information
- Generated stories that include private personal details

Use fictional examples when describing bugs or feature requests.

## Secrets

Do not commit secrets or private credentials.

This includes:

- API keys
- Analytics credentials beyond public tag IDs already present in the site
- Form service private keys
- Deployment tokens
- Private account credentials
- Environment files such as `.env`

If a secret is accidentally committed, remove it from the repository and rotate it immediately.

## Privacy and Child-Safety Concerns

Please report privacy or child-safety concerns privately using the contact email above. Include enough detail to reproduce or understand the issue, but avoid sending real child data unless absolutely necessary.
